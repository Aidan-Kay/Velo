# Vinted Manager — Overview

## Goal

Desktop application for managing a Vinted seller account. Provides listing management, a local item inventory with bulk-listing capability, order tracking, and automatic relisting of sold items.

## Features

| Feature                | Description                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                      |
| ---------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Login**              | Opens a visible BrowserWindow to Vinted for session-based authentication (cookies)                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                               |
| **Dashboard**          | Overview page (default landing). Date range picker (defaults to current month), order stat cards (total revenue, order count, average order value), area chart showing order value over time (uses CSS variables for theme-aware colours), three todo lists: in-stock items without any listing (per-item icon buttons to list immediately or list as draft), drafts awaiting publish (per-item publish icon button), and orders needing user action (rows are clickable — navigate to and highlight the order on the Orders page)                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                               |
| **Listings**           | Pull listings from the authenticated account; sortable table layout with thumbnail, title (clickable link to Vinted), price, favourites (with heart icon), views (with eye icon), status, and actions columns; no default sort (preserves API order); checkbox selection for bulk operations; filter by status and search; skeleton loading indicators when refreshing (all or single listing); per-listing refresh button (uses item_upload detail API); save listing as local Item (with item upload detail API enrichment, including downloading all photos to local disk); green check indicator if already saved as item; update existing item on re-save; publish draft listings via completion API (single-listing refresh after); edit listing button (opens Vinted edit page); duplicate listing action (modal with "Copy photos" and "List as draft" toggles, creates new listing from existing detail); repost active listings (delete + re-create with optional price change and draft mode); bulk repost/publish/save-as-item/delete with progress modal, progress bar, and randomised intervals between operations; bulk delete uses a confirmation modal before starting; bulk save-as-item runs without delay                                                                                                                                    |
| **Items**              | Local draft inventory stored on disk; edit title, description, price and stock; per-item relist toggle (enable/disable automatic relisting); individual list/publish with progress modal (per-photo progress reporting); bulk-list with progress modal (random delays, per-photo progress reporting); bulk edit (modal with optional Price, Stock, Auto-accept fields, each gated by enable checkbox); listing status indicator (Active/Draft/Not listed); "Hide out of stock" checkbox (unticked by default). Items are created from listings only (no create-from-scratch)                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                     |
| **Orders**             | View sold orders in a table with status badges, buyer info (clickable link to Vinted profile), courier (with logo), tracking number (linked with external icon), estimated delivery, shipping label open & download, generate shipping label button with progress pop-up (shows steps, ends with Print/Close buttons, uses preferred label type setting + label_options API fallback, single-order refresh after), per-order refresh button, date+time on separate lines; stage badges with colour coding (purple=label sent, light blue=shipped, green=completed, orange=awaiting pickup, red=cancelled); enriched via transaction detail + journey summary APIs; expandable bundle orders showing individual items; courier-specific label cropping (InPost Locker uses rotation, InPost Home uses portrait crop, Evri uses different crop box); cropped labels sent to configured printer; table checkboxes with bulk actions (bulk generate shipping labels with progress pop-up); automatic stock reduction when order reaches "shipped" stage (configurable in settings); automatic shipping label generation for new orders (configurable in settings); replenish stock action for cancelled orders (disabled once performed); manual mark-as-packed/unpacked action with row-level packed indicator (icon next to stage badge), persisted across polling |
| **Purchases**          | View purchases (orders where user is the buyer) in a table with status badges, seller info (clickable link to Vinted profile), courier (with logo), tracking number (linked), estimated delivery, expandable bundle orders. Background-polled with configurable interval. Virtual scrolling, sort, and filter by status                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                          |
| **Offers**             | View received offers on active listings. Detected by scanning inbox conversations for offer_request_message entities. Background-polled with configurable interval. Accept or counter-offer actions (counter-offer sets local "countered" status with amber badge, preserved across polling), auto-accept at configurable percentage threshold (global in settings, per-item override in edit modal), plus rule-based auto-accept for matching bundles configured on the Automations page. Ignore offer action (local-only, reversible) — sets status to "ignored" to hide from pending view; un-ignore restores to pending. Open conversation action (opens Vinted inbox in browser). Expandable bundle rows with lazy-loaded item details from transaction detail API. Virtual scrolling, sort, and filter by status (defaults to pending)                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                        |
| **Automations**       | Top-level page for managing bundle-offer auto-accept rules. Each rule stores a saved-item tag, an exact bundle item count, and a minimum absolute offer amount; polling auto-accepts when any rule matches and every offered item can be resolved to a tagged saved item by title                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                           |
| **Notifications**      | In-app notification bell in titlebar with unread badge, dropdown list of notifications (new orders, new offers), mark read/clear actions, click-to-navigate with row highlighting, persistence across restarts (max 100), Windows native notifications via Electron Notification API (configurable toggle)                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                       |
| **Relisting**          | Queue-based automatic relisting of sold items after a configurable delay; can list as draft; per-item toggle to enable/disable. Optional smart-relist scheduler defers any due relist until a configured wall-clock time (`HH:MM` local) on the current day; once that time passes, normal cadence resumes                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                       |
| **Activity log**       | In-app viewer for the electron-log `main.log` file. Tail-reads the last ~256 KB, parses each line into `{ ts, level, source, message }`, and renders with level-filter checkboxes, substring search, auto-refresh toggle, and refresh / open-file / clear actions (clear gated by a confirm modal)                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                               |
| **Bulk price rules**   | One-off bulk price reduction on cached active listings older than N days. Settings-managed presets (e.g. `5%/7d`, `10%/7d`) plus a custom percent/days entry; modal shows a debounced dry-run match count, then runs with progress events on `bulk-price-progress` and a final listings refresh                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                  |
| **AI assist**          | Title and description generation in `EditItemModal` via OpenAI (vision chat completions), Ollama (`/api/chat` with `images`), or llama.cpp (OpenAI-compatible). First three item photos are sent; a strict-JSON system prompt yields `{ title, description }`. Disabled until the item is saved and has at least one photo. Confirms before overwriting non-empty fields                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                         |
| **Settings**           | Site selection (styled dropdown), relisting configuration (including smart-relist scheduled start: enable toggle, time input, `5/6/7/8 PM` chips), stock management (reduce stock on shipped toggle), shipping labels configuration (auto-generate labels toggle, preferred label type selector), bulk repost interval configuration (min/max seconds), minimize-to-tray toggle, desktop notifications toggle (`enableNativeNotifications`), label printing configuration (printer and paper size selection from system printers), polling intervals (per-resource configurable: orders, listings, purchases, offers in minutes), auto-accept offers (global percentage threshold with enable/disable toggle), auto-ignore offers (global percentage threshold with enable/disable toggle — pending offers below threshold are locally marked `"ignored"` during polling, no API call), price-rule presets (add/remove preset rules used by Bulk price rules), AI assist (provider chips for OpenAI / Ollama / llama.cpp, key/endpoint/model fields per provider, system prompt textarea)                                                                                                                                                                                                                                                                        |
| **Keyboard shortcuts** | Global shortcuts handled in `App.tsx`: `/` focuses the active page's search input, single-letter keys navigate between pages (`d` Dashboard, `l` Listings, `i` Items, `o` Orders, `p` Purchases, `f` Offers, `a` Automations, `s` Settings, `L` Activity log), `r` triggers the active page's refresh. Suppressed when the focus is in an input/textarea/contenteditable (except `/`); modifier-key combinations are ignored                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                            |

## Tech Stack

| Layer         | Technology                                                                      |
| ------------- | ------------------------------------------------------------------------------- |
| Desktop shell | Electron 29                                                                     |
| UI framework  | React 19 (functional components + hooks)                                        |
| Language      | TypeScript 5.4 (strict mode)                                                    |
| Styling       | Tailwind CSS 4 + shadcn/ui components (Radix UI primitives)                     |
| Icons         | @heroicons/react (Heroicons 2, solid 20px variants)                             |
| Charts        | recharts (AreaChart) + react-is (peer dep)                                      |
| Date utils    | date-fns (date range, formatting, parsing)                                      |
| Date picker   | react-day-picker (shadcn Calendar component)                                    |
| Build         | electron-vite 5 (Vite 8 under the hood; three targets: main, preload, renderer) |
| Type checking | `tsc --noEmit` (electron-vite compiles, tsc only checks)                        |
| Logging       | electron-log (5 MB rotation)                                                    |
| PDF handling  | pdf-lib (label cropping), pdf-to-printer (label printing)                       |
| Packaging     | electron-builder (Windows portable)                                             |

## Data Storage

All persistent data is stored as JSON files in the Electron `userData` directory:

| File                          | Contents                                              |
| ----------------------------- | ----------------------------------------------------- |
| `settings.json`               | Site selection, relisting config, general preferences |
| `items.json`                  | Local item inventory (draft listings)                 |
| `cached-listings.json`        | Cached Vinted listings (from background polling)      |
| `cached-orders.json`          | Cached Vinted orders with enrichment data             |
| `cached-purchases.json`       | Cached Vinted purchases (buyer-side orders)           |
| `cached-offers-received.json` | Cached received offers from inbox scanning            |
| `restock-queue.json`          | Pending relisting entries                             |
| `notifications.json`          | In-app notifications (max 100, newest first)          |
| `window-state.json`           | Window position and size                              |

Additionally, item photos are stored in `userData/item-photos/{itemId}/` as downloaded image files, and debug copies of cropped shipping labels are saved to `userData/cropped-pdfs/`.

## How to Run / Build

```bash
# Install dependencies
npm install

# Development with HMR (renderer) and hot-restart (main/preload)
npm run dev

# Build for production (outputs to out/)
npm run build

# Preview production build
npm start

# Type-check without building
npm run typecheck

# Package for Windows (portable .exe)
npm run build:win
```
